The form-to-report robot

The original form-based intake method, published so any physician can rebuild it in an afternoon: a patient fills a structured form, a small robot turns the answers into a fixed four-section clinical report in your inbox, then deletes its tracks. Two versions below — one on your Google account, one fully local with Ollama — each with a one-click block you can hand to an AI coding assistant to build it for you.

What you need

  • Your own accounts, always. Nothing here is shared: your form, your spreadsheet or computer, your AI key, your professional email.
  • A structured intake form. Google Forms (or any Formsite-class builder that writes rows to a sheet) asking the fixed battery: chief complaint, onset, trigger, location, description, aggravating/relieving, severity, evolution, associated symptoms, treatments tried and effect, chronic conditions, allergies, pregnancy if applicable, anything else.
  • Your own AI API key — stored in Script Properties or an environment variable, never pasted into code. (The local version needs no key at all.)
  • Your professional email address — the report is delivered only there.
  • About 15 minutes with an AI coding assistant, or an afternoon by hand.

How it works

Patient fills the form │ ▼ Row appears in the spreadsheet ──► trigger fires │ ▼ Robot reads the structured fields │ ▼ One AI call fills a FIXED four-section report: 1. Horizontal timeline of the illness (exact dates, day counts) 2. Clinical reasoning decision flow (known / unknown / rule out) 3. Teaching block (pearls, differentials, management, board Q&A) 4. Patient confirmation paragraph — second person, plain words, in the patient's own language, ending with 10 gentle questions │ ▼ Report emailed to the physician ──► row DELETED from the sheet
Two choices carry the method. The output is a template, not an essay — a fixed structure is what makes AI output reviewable at a glance and consistent across hundreds of cases. And the spreadsheet is a conveyor belt, not a chart — the row is deleted once the report is sent; the record of care lives in your chart system, where records belong.

Version 1 — on your Google account

Copy the block below with one click and paste it into an AI coding assistant (built and tested with Claude Code; any capable assistant works) opened anywhere on your computer. It contains the complete instructions and the complete code. The assistant will walk you through the rest and ask only for what must be yours.

THE CODE (install in Apps Script; do not modify the security model):

const CONFIG = Object.freeze({
  API_URL: 'https://api.anthropic.com/v1/messages', // or my provider's endpoint
  API_VERSION: '2023-06-01',
  MODEL: 'claude-haiku-4-5',
  MAX_TOKENS: 8000,
  DOCTOR_EMAIL:
    PropertiesService.getScriptProperties().getProperty('DOCTOR_EMAIL') ||
    'you@your-practice.example',
  SENDER_NAME: 'intake-robot',
});

function getApiKey_() {
  const key = PropertiesService.getScriptProperties().getProperty('AI_API_KEY');
  if (!key) throw new Error('Set AI_API_KEY in Script Properties first.');
  return key;
}

// Map your sheet's columns here (0-based indexes into the row).
const FIELDS = [
  { label: 'Patient ID', index: 3 },
  { label: 'Gender', index: 4 },
  { label: 'Age', index: 5 },
  { label: 'Chief complaint', index: 6 },
  { label: 'Start date', index: 7 },
  { label: 'Trigger', index: 8 },
  { label: 'Location', index: 9 },
  { label: 'Description', index: 10 },
  { label: 'Aggravating', index: 11 },
  { label: 'Relieving', index: 12 },
  { label: 'Severity', index: 13 },
  { label: 'Evolution', index: 14 },
  { label: 'Associated symptoms', index: 15 },
  { label: 'Treatments tried', index: 16 },
  { label: 'Treatment effectiveness', index: 17 },
  { label: 'Chronic conditions', index: 18 },
  { label: 'Allergies', index: 19 },
  { label: 'Pregnant/breastfeeding', index: 20 },
  { label: 'Additional', index: 21 },
];
const LAST_COLUMN = 22;

function setupTrigger() {
  const sheet = SpreadsheetApp.getActiveSpreadsheet();
  ScriptApp.getProjectTriggers().forEach((t) => ScriptApp.deleteTrigger(t));
  ScriptApp.newTrigger('onNewRow').forSpreadsheet(sheet).onChange().create();
  SpreadsheetApp.getUi().alert('Enabled', 'Auto-processing active.',
    SpreadsheetApp.getUi().ButtonSet.OK);
}

function onNewRow(e) {
  try {
    if (e?.changeType !== 'INSERT_ROW') return;
    const sheet = SpreadsheetApp.getActiveSheet();
    const lastRow = sheet.getLastRow();
    if (lastRow < 2) return;
    const rowData = sheet.getRange(lastRow, 1, 1, LAST_COLUMN).getValues()[0];
    if (!rowData || rowData.length < LAST_COLUMN) throw new Error('Invalid row');
    sendEmail(callModel(createPrompt(rowData)), rowData);
    // Data minimization: the pipeline keeps nothing. The chart lives in
    // your record system, not in this spreadsheet.
    sheet.deleteRow(lastRow);
  } catch (error) {
    console.error('Error:', error);
  }
}

function createPrompt(rowData) {
  return FIELDS.map(({ label, index }) =>
    `${label}: ${rowData[index] ?? ''}`).join('\n');
}

function getSystemPrompt() {
  // Built with the assistant in step 5 — the fixed 4-section template.
  return 'Output ONLY valid HTML starting with <!DOCTYPE html>. ' +
    'Generate a report with EXACTLY these 4 sections... (template here)';
}

function callModel(prompt) {
  const response = UrlFetchApp.fetch(CONFIG.API_URL, {
    method: 'post',
    headers: {
      'Content-Type': 'application/json',
      'x-api-key': getApiKey_(),
      'anthropic-version': CONFIG.API_VERSION,
    },
    payload: JSON.stringify({
      model: CONFIG.MODEL,
      max_tokens: CONFIG.MAX_TOKENS,
      temperature: 0.1,
      system: getSystemPrompt(),
      messages: [{ role: 'user', content: prompt }],
    }),
    muteHttpExceptions: true,
  });
  const result = JSON.parse(response.getContentText());
  if (result.error) throw new Error(result.error.message);
  return (result.content?.[0]?.text ?? '')
    .replace(/```html\n?/g, '').replace(/```\n?/g, '').trim();
}

function sendEmail(htmlContent, rowData) {
  GmailApp.sendEmail(
    CONFIG.DOCTOR_EMAIL,
    `Intake report for "${rowData[3] ?? 'Unknown'}"`,
    'See HTML.',
    { htmlBody: htmlContent, name: CONFIG.SENDER_NAME }
  );
}

function onOpen() {
  SpreadsheetApp.getUi().createMenu('IntakeRobot')
    .addItem('Setup', 'setupTrigger').addToUi();
}

What a consultation actually costs

Prices move — check the providers' pages. As of mid-2026, with a typical consultation of roughly 2,500 input tokens and 4,000 output tokens:

PathApprox. pricePer consultationWhen to choose it
Claude Haiku 4.5 on AWS Bedrock ~$1 / M in · ~$5 / M out ≈ 2–3 cents Choose a BAA-covered route whenever identifiable patient data is involved in a jurisdiction with a compliance framework. Bedrock is one worked example of that category: a signed BAA (the HIPAA path in the US), managed keys, audit logging, regional data residency. Equivalent covered offerings from other clouds serve the same purpose.
Claude Haiku 4.5, direct API same ≈ 2–3 cents Same model without the compliance wrapper — for de-identified or educational use.
DeepSeek (current chat model) ~$0.28 / M in · ~$0.42 / M out ≈ a quarter of a cent One example of the low-cost tier that fits education-mode and de-identified use, and the multi-AI panel's second and third seats. Roughly 10× cheaper; no BAA. Comparable budget models exist from several providers.
Ollama, fully local (below) free after download ≈ 0 — electricity only Nothing leaves your machine at all. The right choice where privacy must be absolute or connectivity is unreliable.
Read it two ways. Practically: choose by data sensitivity, not price — with identifiable data, the ~10× premium for the BAA-covered path is three cents well spent. Morally: even the expensive path is pennies. This is the number behind our claim that leaving the poorest without structured guidance is a choice, not a resource problem.

The security basis of this system

Built into the design

  • No third-party server of ours. Your form, your spreadsheet or computer, your script, your email — the only outside party is the AI provider you chose (and none at all in the local version).
  • Data minimization by construction. The row is deleted after the report sends; the pipeline accumulates nothing that could be breached later. The record of care lives in your chart system.
  • One recipient. Reports go only to the physician. No machine ever messages a patient in this pipeline.
  • A fixed template resists prompt games — and makes drift or tampering obvious on review.

Your responsibilities — the part no script can do

  • Never hardcode an API key. A key in code is one paste, one screen share, one repository push away from public — and a leaked key lets strangers spend on your account. Keys live in Script Properties or environment variables, and are treated as burned the moment they appear anywhere else. Rotate immediately if exposed.
  • Know where patient text travels: form → the form host → the AI provider → your inbox. Handling identifiable data in a covered jurisdiction means covering each hop (e.g. a Google Workspace agreement, plus a BAA-covered AI path). Can't cover a hop? Then run intake on a case number and join identity only inside your chart system.
  • Restrict the form and the sheet. The link is for patients; the sheet and the script are yours alone.
  • Email is the weakest hop. The inbox that receives reports needs strong authentication; it briefly holds clinical content.
  • Test with fake patients only, and keep yourself in the loop: the report is a draft for your judgment, never an answer sent onward by a machine.

Version 2 — fully local, with Ollama (nothing leaves your machine)

The same conveyor-belt pattern, with every hop on your own computer: a local intake page instead of a hosted form, a folder of reports instead of email, and a local model instead of a cloud API. No key, no account, no internet needed once the model is downloaded. Copy the block and hand it to your AI coding assistant:

Copied.

You are my coding assistant. Build me the fully LOCAL version of the
FreeEducationHealth "form-to-report robot": an intake pipeline that runs
entirely on my own computer with Ollama, so no patient text ever leaves
this machine. Walk me through every step; assume I do not code.

STEPS YOU WILL DO WITH ME:
1. Install Ollama (https://ollama.com — Windows/Mac/Linux installers) and
   pull a capable general model, e.g.:  ollama pull llama3.1:8b
   (If my computer is strong, suggest a larger model; if weak, a smaller
   one. Explain the tradeoff simply.)
2. Install Node.js LTS if missing (https://nodejs.org).
3. Create a folder called intake-robot with the single file below
   (server.cjs), then run:  node server.cjs
4. Open http://localhost:8787 on this computer (or a tablet on my own
   network) — that is the patient intake page. Each submission becomes a
   file in reports/ named by case number and time, containing the fixed
   four-section report. Nothing is emailed and nothing goes online.
5. Help me adapt the intake questions and the report template to my
   language and my practice (same 4 sections as the Google version:
   timeline, reasoning flow, teaching block, and a second-person
   plain-language confirmation paragraph ending in 10 gentle questions).
6. Test with an OBVIOUSLY FAKE patient first. Remind me: reports/ holds
   clinical drafts, so it stays on this machine, disk encryption on, and
   the page is for my own network only — never exposed to the internet.

THE CODE (server.cjs — single file, no dependencies beyond Node 18+):

'use strict';
const http = require('http');
const fs = require('fs');
const path = require('path');

const PORT = 8787;
const OLLAMA_URL = 'http://localhost:11434/api/chat';
const MODEL = process.env.INTAKE_MODEL || 'llama3.1:8b';
const REPORTS_DIR = path.join(__dirname, 'reports');
if (!fs.existsSync(REPORTS_DIR)) fs.mkdirSync(REPORTS_DIR);

const QUESTIONS = [
  ['caseId', 'Case number (not a name)'],
  ['gender', 'Gender'],
  ['age', 'Age'],
  ['complaint', 'What is the main problem?'],
  ['start', 'When did it start? (date)'],
  ['trigger', 'Did anything seem to set it off?'],
  ['location', 'Where in the body is it?'],
  ['description', 'Describe how it feels'],
  ['aggravating', 'What makes it worse?'],
  ['relieving', 'What makes it better?'],
  ['severity', 'How bad is it, 0 to 10?'],
  ['evolution', 'Getting better, worse, or stable?'],
  ['associated', 'Any other symptoms with it?'],
  ['treatments', 'What have you tried?'],
  ['effect', 'Did that help?'],
  ['conditions', 'Any long-term health conditions?'],
  ['allergies', 'Any allergies?'],
  ['pregnancy', 'Pregnant or breastfeeding? (if applicable)'],
  ['additional', 'Anything else to add?'],
];

const SYSTEM_PROMPT = [
  'You draft a clinical intake report for a physician. Output plain HTML',
  'starting with <!DOCTYPE html>, with EXACTLY four sections:',
  '1. Horizontal text timeline of the illness with dates and Day N markers',
  '   (text characters only).',
  '2. Clinical reasoning decision flow: what we know / what we need /',
  '   must rule out / decision / suggested next steps for the PHYSICIAN.',
  '3. Teaching block: 3 diagnostic pearls, 3 hallmarks, 3 differentials',
  '   with distinguishing clues, evidence-based management, follow-up.',
  '4. A confirmation paragraph addressed to the patient in plain everyday',
  '   words, second person, starting "Just to confirm: you are..." and',
  '   ending with 10 gentle follow-up questions.',
  'This is a DRAFT for a physician to review. Never invent facts not in',
  'the intake. Flag any red-flag symptoms prominently at the top.',
].join('\n');

function formPage() {
  const fields = QUESTIONS.map(([k, label]) =>
    `<label>${label}<br><input name="${k}" style="width:100%"></label><br><br>`
  ).join('');
  return `<!doctype html><html><head><meta charset="utf-8">
<meta name="viewport" content="width=device-width,initial-scale=1">
<title>Check-in</title></head>
<body style="font-family:sans-serif;max-width:640px;margin:2rem auto;padding:0 1rem">
<h2>Check-in</h2>
<p>Please answer what you can. Do not type your name or ID numbers —
your case number is enough. This prepares your visit; it is not a
diagnosis, and in an emergency seek help immediately.</p>
<form method="POST" action="/submit">${fields}
<button style="padding:.7rem 1.4rem">Send</button></form></body></html>`;
}

async function draftReport(answers) {
  const intake = QUESTIONS.map(([k, label]) =>
    `${label}: ${answers[k] || ''}`).join('\n');
  const res = await fetch(OLLAMA_URL, {
    method: 'POST',
    headers: { 'content-type': 'application/json' },
    body: JSON.stringify({
      model: MODEL, stream: false,
      messages: [
        { role: 'system', content: SYSTEM_PROMPT },
        { role: 'user', content: intake },
      ],
    }),
  });
  if (!res.ok) throw new Error('Ollama error HTTP ' + res.status);
  const data = await res.json();
  return (data.message && data.message.content) || '';
}

const server = http.createServer(async (req, res) => {
  if (req.method === 'GET' && req.url === '/') {
    res.writeHead(200, { 'content-type': 'text/html; charset=utf-8' });
    return res.end(formPage());
  }
  if (req.method === 'POST' && req.url === '/submit') {
    let body = '';
    req.on('data', (c) => { body += c; if (body.length > 1e6) req.destroy(); });
    req.on('end', async () => {
      try {
        const answers = Object.fromEntries(new URLSearchParams(body));
        const caseId = (answers.caseId || 'case').replace(/[^a-zA-Z0-9_-]/g, '');
        const stamp = new Date().toISOString().replace(/[:.]/g, '-');
        const report = await draftReport(answers);
        fs.writeFileSync(path.join(REPORTS_DIR, `${caseId}_${stamp}.html`),
          report, 'utf8');
        res.writeHead(200, { 'content-type': 'text/html; charset=utf-8' });
        res.end('<p style="font-family:sans-serif">Received — thank you. ' +
          'Your clinician will review it with you.</p>');
      } catch (e) {
        res.writeHead(500, { 'content-type': 'text/plain' });
        res.end('Something went wrong; please tell the clinic. ' + e.message);
      }
    });
    return;
  }
  res.writeHead(404); res.end('Not found');
});

server.listen(PORT, () => {
  console.log(`Intake robot: http://localhost:${PORT}`);
  console.log(`Model: ${MODEL} via ${OLLAMA_URL}`);
  console.log(`Reports are written to: ${REPORTS_DIR}`);
  console.log('Local use only — do not expose this to the internet.');
});
Local rules. The intake page is for your own device or clinic network only — never exposed to the internet. The reports/ folder holds clinical drafts: keep disk encryption on, and treat the machine like the filing cabinet it now is. The patient never sees the report; a clinician reviews every draft. And the same line as always: this drafts and prepares — it does not diagnose, treat, or replace a physician.
