DeckStation

An always-on-top dock of buttons that floats above every window — Stream Deck software without the $200 hardware. One click pastes a stored text into whatever app you are working in. For a clinician using the template method, that means press-SOAP, press-Prescription, press-Referral — straight into your chatbot. It ships with the clinical template pack already loaded as a "Clinic Templates" profile.

What it does

  • Six button types: open a website, launch an app or protocol, run a program, send a hotkey, and — the one clinicians use most — paste a stored text (up to 5,000 characters) into the focused window with one click. The dock never steals focus; your chatbot stays active.
  • Multiple named profiles — the bundled "Clinic Templates" profile carries the full template pack: context control, SOAP, prescriptions, labs, referral, imaging, two work-leave variants, summary, timeline, decision flow, and the #7 patient paragraph.
  • Stream Deck import — profiles already installed in the official Stream Deck software are detected and converted.
  • Show/hide from anywhere — double-tap Left Alt, or Ctrl+Alt+Space / Ctrl+Alt+D, or the tray icon.
  • No account, no server, no telemetry. Your buttons live in one local file (%APPDATA%\dockstation\config.json). The only network listener is a local-only bridge (127.0.0.1) for the optional Stream Deck plugin.

Known bugs — read this before relying on it

This tool is offered free and honestly: it works, and it has real rough edges. This list comes from a full multi-model audit run before publication, written so you know exactly what you are getting.

The paste button types into whatever window has focus. There is a short delay between click and paste, and no check of where the text lands. If focus shifts in that gap — a popup, an alt-tab, a misclick — your canned text lands in the wrong app or the wrong patient thread. Click into the exact input box first, watch the paste land, and always review before sending. Never use it blind.
  • Clipboard notes: the paste works through the Windows clipboard, so the pasted text stays on your clipboard afterward — and Windows clipboard history (Win+V) or cloud clipboard sync may retain it. On a shared machine, clear it after use. In rare cases under load the previous clipboard contents can paste instead — one more reason to watch every paste land.
  • First-click misses: after moving your mouse onto the dock, the first click or two can be eaten by the overlay's click-through logic. If a button doesn't fire, click again.
  • Overlay glitches: rarely, the invisible overlay can wedge into a state where it eats clicks (desktop feels frozen) or the dock is invisible while logically shown. Toggling with Ctrl+Alt+Space or restarting from the tray recovers it.
  • Hotkey footprint: the app registers Ctrl+Alt+Space and Ctrl+Alt+D system-wide and watches for double-Alt taps (it inspects only the Alt key — no keylogging). A quick double Alt in any app can toggle the dock, and after remote-desktop or sleep the double-Alt toggle can silently die (the Ctrl+Alt shortcuts keep working).
  • Electron 29 base: the app runs on an older Electron with known CVEs; it loads no remote content, which limits practical exposure, but an upgrade is on the roadmap.
  • Back up your buttons: your whole configuration is one file — %APPDATA%\dockstation\config.json. Copy it somewhere safe once your buttons are set up.

If it breaks, have an AI assistant fix YOUR copy

DeckStation is distributed source-included on purpose. If something misbehaves, open its folder in an AI coding assistant (built and tested with Claude Code; any capable one works) and describe the problem in plain words — "the dock stopped staying on top, fix it" or "make the paste button confirm the target window first." The assistant reads the code and repairs your copy. That is not a workaround; it is the intended way to own software like this — the same philosophy as everything else on this site.

Get it

1

Download

The free bundle (portable app + full source + the Clinic Templates profile) is on the download page. No installer needed for the portable version — one .exe you run directly.

2

First run

Launch it, and switch the profile selector to "Clinic Templates". Windows SmartScreen may warn on an unsigned free tool — that is expected for unsigned software; proceed only if you downloaded it from this project's official page.

3

Use it with the template method

Open your chatbot, click into its input box, and drive your documentation one button at a time: the template method.

Scope note. DeckStation itself holds no patient data unless you put patient text into a button — don't. Buttons are for reusable templates; the patient specifics live in your chatbot conversation and your chart system, under the privacy rules of the template method.
