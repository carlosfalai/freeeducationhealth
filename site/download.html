Prefer to build the zips yourself from a checkout of the repo? Run node package-releases.cjs and the same bundles are written to dist/, fresh from the current source each run. It needs only Node — no npm install and no external zip utility.

Rebuild this website if it goes down

This whole site is plain static HTML. There is no build step, no framework, and no external dependency — one stylesheet and a few pages that work when opened straight from a folder. Every download above already includes the site/ folder, so a copy of this website travels with the project.

If the original site is ever unavailable, taken down, or attacked, anyone holding a copy can bring it back. Re-hosting it is just copying a folder to any static host.

1

Get the site/ folder

Take it from the full download or from a checkout of the repository. It is a complete website on its own — no other files are required.

2

Host it as-is

Point a static host at the folder. It deploys unchanged to Cloudflare Pages: create a Pages project, set the build output to the site/ directory, and leave the build command empty. Any other static host works the same way, and you can even open index.html by double-clicking it locally.

3

Share the copy

Because there is no central server and nothing to configure, the copy you host is a full, working mirror. Health information that cannot be quietly switched off is the whole point.

# Cloudflare Pages: no build command, output = the site folder Build command: (leave empty) Build output dir: site # Or open it with no host at all: open site/index.html
What's guaranteed clean. Every bundle is built only from version-controlled files, so credentials, patient data, generated PDFs, and signature images are never included. Anything git ignores stays out of the zip, and a final safety check re-verifies each file before it is packaged.
